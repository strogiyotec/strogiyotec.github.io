Request and database domains

May 21 , 2020

Let me show you an example from my plain REST API implementation called widrest but before you have to understand business logic. The main domain of the business is called Widget. Widget has some properties namely:

  • Y coordinate
  • X coordinate
  • Z index
  • Last updated date
  • Width
  • Height
  • Database id

Simple enough to understand, and now we have the following requirement:
User can specify all attributes except for Id and Last Updated time through web form and as a response server saves this data, assign unique identifier to widget and set last updated time to creation date.User can assign null value for z-index , in this case server will assign the value lower than the minimum z-index in database on 1 Again quite easy task
If you asked me to implement it one year ago I would make a simple domain for Database layer and Request form called Widget but not now and here are some problems with this implementation
